# CourseIntel — Frontend Screen Inventory

All screens are implemented as Next.js 14 App Router pages under `frontend/app/`.

---

## Authentication Screens (`frontend/app/(auth)/`)

| Screen | File | Description |
|--------|------|-------------|
| **Login** | `(auth)/login/page.tsx` | Email + password login via Supabase Auth. Redirects to `/dashboard` on success. |
| **Sign Up** | `(auth)/signup/page.tsx` | New account registration. Sends verification email. |
| **Email Verify** | `(auth)/verify/page.tsx` | Prompts user to check inbox; resend button included. |
| **Demo** | `(auth)/demo/page.tsx` | One-click demo access that loads the CSC 212 fixture without a real account. |

---

## Dashboard Screens (`frontend/app/dashboard/`)

| Screen | File | Description |
|--------|------|-------------|
| **Dashboard Home** | `dashboard/page.tsx` | Course list with GPA summary card, extension promo banner, and quick-add course button. |

---

## Course Screens (`frontend/app/dashboard/course/[id]/`)

All course screens share a course context fetched once via `hooks/use-course.ts` (SWR).

| Screen | File | Route | Description |
|--------|------|-------|-------------|
| **Course Profile** | `page.tsx` | `/dashboard/course/:id` | Bootstrap status stepper (7 agents), detected tools, student signal (workload hours, difficulty, common pitfalls). |
| **Setup / Bootstrap** | `setup/page.tsx` | `/dashboard/course/:id/setup` | Agent health badge, form to trigger the 7-step bootstrap pipeline. Shows live progress per agent step. |
| **Grades** | `grades/page.tsx` | `/dashboard/course/:id/grades` | Grade entry table, weighted GPA bar (`GradeHero`), category breakdown chart (`CategoryBreakdown`), manual entry form. |
| **Goals** | `goals/page.tsx` | `/dashboard/course/:id/goals` | Target letter grade selector, required-score calculator (`GoalSimulator`), feasibility indicator. |
| **Study** | `study/page.tsx` | `/dashboard/course/:id/study` | Study material upload (PDF/notes), extracted content display, study plan generation via `StudyPlanAgent`. |
| **Resources** | `resources/page.tsx` | `/dashboard/course/:id/resources` | Curated external resources (GitHub repos, open courseware, Reddit threads, YouTube) sourced by `PublicResourcesAgent`. |
| **Actions** | `actions/page.tsx` | `/dashboard/course/:id/actions` | Urgency-ranked obligation list (deadlines, exams, projects), ICS calendar import, action plan generated by `JudgmentAgent`. |

## Navigation Model

```
/                          ← Landing page (public)
/login                     ← Auth
/signup                    ← Auth
/verify                    ← Auth
/demo                      ← Demo (no account needed)
/dashboard                 ← Protected (requires auth)
/dashboard/course/[id]             ← Course profile
/dashboard/course/[id]/setup       ← Bootstrap trigger
/dashboard/course/[id]/grades      ← Grade tracker
/dashboard/course/[id]/goals       ← Goal simulator
/dashboard/course/[id]/study       ← Study materials
/dashboard/course/[id]/resources   ← External resources
/dashboard/course/[id]/actions     ← Action board
```

Authentication is enforced via `frontend/middleware.ts` (Supabase session check on all `/dashboard/*` routes).

## Demo Fixture

The demo mode (`/demo`) loads `frontend/public/demo/csc212-uri-bootstrap-demo.json` — a fully populated
bootstrap result for CSC 212 (Data Structures) at the University of Rhode Island. This allows judges
and evaluators to explore all 7 course screens without an LLM API key or Supabase account.

The fixture includes:
- 6 grading categories with weights summing to 1.0
- 8 curated external resources (GitHub, Reddit, YouTube, open courseware)
- 5 urgency-ranked obligations (midterm, project, final)
- Student signal: workload hours, difficulty, common pitfalls
- Detected tools: Gradescope, Ed Discussion, URIx LMS
